6. Designation and Role Mapping

Automate your security workflow by linking job titles to permission groups. This mapping ensures that users inherit the correct access rights the moment their designation is assigned.

Purpose

Map organizational Designations (job titles) to one or more Roles (permission groups). This ensures that when a user is assigned a designation, they automatically inherit the correct access rights without manual intervention.

Navigation:
Menu Designation and Role Mapping

Main Features

Multi-select Chip Inputs

Add multiple items; each appears as a removable “pill/chip” for clean organization.

Search & Autocomplete

Type-to-find functionality ensures rapid selection from large datasets.

Inline Editing

Manage mapping rows directly within the table view for maximum efficiency.

Group Logic

Map multiple designations to a single role (or vice versa) in one unified row.

How to Use

NEW MAPPING
1

Designations Field: Type a job title and select it from the suggestions. Repeat for multiple titles.

2

Role Field: Type and select the role(s) to assign to those designations.

3

Save: Click the Save/Update button to activate the mapping.

UPDATE MAPPING
Expand: Click into the field and type to append new designation chips.
Refine: Tap the small × on any specific chip to detach it from the role.
REMOVE WHOLE ROW

Wipe all active chips from both fields in the specific row.

Click the Trash icon and commit with Save to purge the mapping.
